# Proyecto - Resumen

## Problema que resuelve

En las universidades, las oportunidades para estudiantes (tutorías, grupos de estudio, ayudantías, trabajos part-time, prácticas, voluntariados, investigación) están dispersas en correos, grupos de WhatsApp, redes sociales y murales. Esto hace que muchas oportunidades no lleguen a quienes podrían aprovecharlas, y que los publicadores no tengan una forma simple de difundirlas y gestionar postulaciones.

## Objetivo

Desarrollar **CampusLink**, una plataforma web que centralice oportunidades universitarias y facilite la conexión entre quienes publican y quienes buscan participar.

## Alcance del proyecto

El alcance del proyecto cubre:

1. **CRUD de oportunidades** (crear, listar, buscar, ver detalle, editar, eliminar)
2. **Autenticación** de usuarios (registro + login con JWT)
3. **Roles de usuario** (estudiante, docente, institución, administrador)
4. **Sistema de postulación** a oportunidades
5. **Panel del publicador** para gestionar postulantes
6. **Notificaciones** básicas

Para **Entrega 1**, el MVP implementa los puntos 1 y 2.

## Usuarios objetivo

| Tipo de usuario | Descripción |
|---|---|
| Estudiante | Busca y postula a oportunidades |
| Docente | Publica tutorías, proyectos de investigación |
| Institución | Publica prácticas, voluntariados, trabajos |
| Administrador | Gestiona la plataforma y los usuarios |

## Misión

Facilitar el acceso oportuno a oportunidades universitarias mediante una plataforma centralizada, simple y útil para la comunidad.

## Visión

Ser el principal punto de encuentro digital entre estudiantes, docentes e institución para descubrir y gestionar oportunidades dentro del entorno universitario.
